STEP 1: How to Add a Word

In order to add a word to begin translation we need to open PostMan to test our POST action on Words.

Once on PostMan follow these steps

1). From the Drop Down Arrow Change Action to POST and Change select from Authorization to Body

2). Enter Address : 127.0.0.1:8948/words

3). if form-data is bubbled in, change it by selecting raw

4). Make Sure the drop down arrow to the right of binary bubble is JSON(application/json) and not text.

5). Inside the API response content enter : { "word" : "Soccer" }

5.1). Note "Nothing" can be changed to any word you would like to add.

5.2). { "word" : "ENTER YOUR ENGLISH WORD HERE" }

6). Press Send for API request.

6.1). if you want to see changes of your word being added : From the Drop Down Arrow Change POST to GET

STEP 2: How to Add a Translated Word

In order to add a Translated Word keep PostMan Open to test our POST action on Targets.

1). From the Drop Down Arrow Change Action to POST and Change select from Authorization to Body

2). Enter Address : 127.0.0.1:8948/targets

3). if form-data is bubbled in, change it by selecting raw

4). Make Sure the drop down arrow to the right of binary bubble is JSON(application/json) and not text.

5). Inside the API response content enter : { "target": "Futbol", "wordId": 3, "languageId": 2 }

5.1). Where "target" : "Enter Translated Word" , is a translated word

5.2). So for my example, eariler when we added a word, i choose to add the enlish word Soccer and the Spanish Translation is Futbol.

5.3). You will add the spanish translation of the word you choose above

5.4). "wordId": 3 , eariler when you added a word, that actually became the 3rd word in my data base.

5.5). so if you only added one word on STEP 1, wordId will be 3.

if you added two words, then the first word you added will be "wordId": 3, and you will follow STEP 2 again to account for the second word you added which will be "wordId": 4.

6.1). "languageId": 2 will always stay constant no matter how many words you added. because we are doing English to Spanish

6.2). "languageId": 2 means we are adding the Spanish Translation, 1 is the languageId of English.

6.3). the important thing to remember here is my database starts initialized with two words and corresponding translations for those words so wordId has to start at 3 and move up from there.

6.3). just to make sure everything is clear. if you added cat and dog, when you add the translated word for cat, your "wordId": 3 and the next API response for translated word for Dog will be "wordId": 4

127.0.0.1:8948/targets/getTranslationByLanguageId?word= ENTER YOUR WORD HERE &languageId=1&targetLanguageId=2

for my example my word is Soccer but you will enter your word, &languageId=1 is saying this is a english

&targetLanguageId=2 is telling us to do a spanish translation for the given word.

the url can either be entered in Chrome or PostMan

2. IMPORTANT - if you want to do Spanish to English switch &languageId to 2 and &targetLanguageId to 1.

Get Translation By Language

After you completed Step 1 and 2 we are to test our Translation ENTER THE URL :

127.0.0.1:8948/targets/getTranslationByLanguage?word=Soccer&language=English&targetLanguage=Spanish

127.0.0.1:8948/targets/getTranslationByLanguage?word= ENTER YOUR WORD HERE &language=English&targetLanguage=Spanish

For my example my word is Soccer but you will enter your word, &language=English is is saying this is a english

&targetLanguage=Spanish is telling us to do a spanish translation for the given word.

IMPORTANT if you want to do Spanish Word to English, On Step 1: added a spanish word,follow STEP 2: but "languageId": will be 1 ("languageId":1). and when you get here to doing the actual translation switch &language and &targetLanguage.
